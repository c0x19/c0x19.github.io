0x04 编码转换

ASCII编码

ASCII码是美国信息交换标准码，是一种标准的单字节字符编码方案，是最基础的计算机识别码，将单字节字符与数字进行一一对应，便于计算机识别

http://ascii.911cha.com/

进制转换

日常用的进制为10进制，计算机一般是16进制和二进制，网上有很多在线进制转换工具

https://tool.oschina.net/hexconvert

base编码

base64就是基于64个可打印字符来表示二进制数据的一种方法

实现：一个字节为8位，我们只需要其中的6位即可（2的6次方为64）

可见，按顺序依次6位来编码即可，但会出现编码比特位数不能被6整除，因此需要填充，我们用0来填充使得变为6的倍数，并在编号后加上 =号来表示填充了多少0（一个等号代表2个0）

注：

  • base32就是使用32个基本字符
  • base编码特点是只会出现基本字符，并且一般末尾都会有=号
  • base64结尾最多有2个=号
  • base32结尾最多有6个=号

URL 编码

也叫做百分号编码，是网页上应用的编码，特点是%

https://tool.oschina.net/encode?type=4

Unicode编码

为了将所有的字符都表示出来，1字节的ascii显示是不够的，因此有了2字节表示一个字符的unicode编码

有四种格式：

  • &#[hex] &#0054
  • &#[dec] &#00084
  • \u[hex] \u0054
  • \u+[hex] \u+0054

Morse编码

也就是广为人知的摩斯密码

  • 只有.和-
  • 一点的长度时一个单位，一划是三个单位
  • 间隔
    • 一个字母中点划之间的间隔是一点（一个单位）
    • 两个字母之间的间隔是三点（一划）
    • 两个单词之间的间隔是七点

Jother编码

俗称JSFuck，使用两个字符：[]()!+来编写JavaScript程序

0x05 隐写

隐写一般分为两种类型，插入和替换

  • 插入：利用文件格式的无光数据或空白区域放置需要的数据，不会改变原始数据
  • 替换，如LSB替换，把每个字节最低有效位变换，不会改变文件大小

图片尾部插入

  • 出题方法：使用16进制编辑器打开图片。在末尾添加特殊字符串，如flag

  • 插入一般是JPG图片，JPG图片的结尾都是FF D9

使用010 edit打开即可发现在结尾符FF D9之后是flag，有时候拿到的是base编码后的flag，需要再解码一下

图片中间插入

JPG图片会有标记码，标记码高字节固定为0xFF，标记码之间有冗余字节，在中间插入隐秘数据即可，不会影响标记码位置，图片可正常打开

同上，容易找到flag

这两种一般可以直接用字符串搜索功能来寻找隐秘信息

插入压缩包

可以在图片中插入压缩文件，利用windos命令行工具即可实现插入

copy /b old.jpg+flag.zip new.jpg

最简单的办法就是修改后缀名为.zip，然后解压即可

但最正确的还是利用binwalk来分析

明显可以看到图片里包括一个zip文件，里面有flag.txt

使用foremost来将图片分离输出到flag文件夹中

foremost new.jpg -o flag

插入图片

可以在图片中再插入图片

同上

copy \b old.jpg+flag.jpg new.jpg

同上使用binwalk分析即可

LSB隐写

LSB是最低有效位的英文缩写

LSB隐写只适用于png和bmp文件，jpg是有损压缩，压缩过程中或破坏信息

原理：LSB隐写就是修改图片中像素的三原色最低位，肉眼是看不出区别的

修改最低位bit，使用工具Stegsolve处理LSB隐写题

例题1

使用Stegsolve工具选中三原色最低位

拿到了加密信息 9keKZ9LDcKHV@@)-=UD)AN2PSBiAA[0OJ=.DeslN

是一个base85编码，解码即可

例题2

使用工具打开图片，点击下面的移动按钮，修改到gray bits，即发现了二维码，扫码拿到了flag

Exif隐写

jpg图片比其他图片多了exif描述，这种题非常简单，查看 图片属性即可

PNG

PNG文件，文件头是由固定的字节来描述的，剩余的部分由三个以上的数据块特定顺序组成，其中有4个关键数据块

  • 文件头：89 50 4E 47 0D 0A 4A 0A

  • 数据块

    | 符号名 | 名称 | 数量 | 必须/可选 | 位置限制 |
    | —— | ————– | —- | ——— | ————– |
    | IHDR | 文件头数据块 | 1 | 必须 | 第一块 |
    | PLTE | 调色板数据块 | 1 | 可选 | IDAT之前 |
    | IDAT | 图像数据块 | 多个 | 必须 | 与其他IDAT连续 |
    | IEND | 图像结束数据块 | 1 | 必须 | 最后一个数据块 |

其中每个数据块都有统一的数据结构，每个数据块由4部分组成：

  • 长度：4字节，数据块中数据域的长度
  • 类型：4字节
  • 数据：可变长度，具体数据
  • CRC：4字节，循环冗余检测，是通过类型和数据部分进行计算得到的

IHDR块隐写

PNG图片显示的大小是由IHDR块来控制

打开图片看到提示flag在本图片下面

使用010编辑器打开，修改增大图片的长度

保存后查看图片，发现flag

linux环境变量

Posted on 2019-08-09

0x01 什么是环境变量

程序和脚本通过环境变量来获取

  • 系统信息
  • 存储临时数据
  • 配置信息

环境变量分为两种：

  • 全局变量：对于shell及它的子shell都可见
  • 局部变量：只shell自己可见

查看环境变量

  • 查看全部全局变量
    • printenv
    • env
    • set
      • 可以查看全局变量、局部变量、用户定义的变量
  • 查看单一环境变量的值
    • printevb HOME
    • echo $HOME

设置用户定义变量

  • 创建局部变量: my_var="hello world"
    • 用户定义的环境变量使用小写，区分系统的大写环境变量
    • 环境变量值中有空格要用双引号
  • 创建全局变量
    • my_var="hello world"
    • export my_var
  • 删除环境变量
    • unset my_var

0x02 PATH环境变量

环境变量中有一个PATH环境变量

  • 存储命令和程序查找的目录

  • 每个目录用冒号隔开

  • 方便程序在任意目录中快速执行，而不必进入其目录下才能运行

添加新目录

PATH=$PATH:/home/nik/test

通常运行一个程序需要这样：./test

我们可以将单点符加入PATH环境变量来方便运行程序

  • PATH=$PATH:.
  • test

0x03 持久化存储环境变量

在~/.bashrc文件中添加或修改环境变量即可

更新生效

source ~/.bashrc

编码

Posted on 2019-04-23

编码发展史

1.ANSI

最初计算机只有一种字符集，即ASCII字符集，最初只用7bit表示一个字符，后来扩展成8bit

2.MBCS

随着各国语言加入，ASCII编码不够用，每个国家都有了自己的字符集编码，在ASCII的基础上扩充，兼容ASCII 127，如我国的GB-2312，用2字节来表示一个字符

3.Unicode（UTF-16）

多种编码字符集混杂，国际交流每次都要转码，很不方便，因此发展出了Unicode编码，固定用2个字节来表示一个字符，可以表示世界上所有的语言字符

4.UTF-8

Unicode每次都用2个字节，浪费空间，传输大量数据时浪费流量，因此有了UTF-8这种变长的编码

哪个好?

不要问原因，用UTF-8就行了

gdb查看内存

Posted on 2019-04-23

数据在内存中存储形式

不论什么类型的数据，在机器中都是以0或者1，即bit来存储的，而有了内存后，最小的存储数据单元是字节，1byte=2hex=8bit

存储类型

不论什么类型，在内存中实际都是16进制存储的数据，根据类型的不同显示出不同的结果，来给人看。所有数据都是16进制，不同的是字节长度

类型转换

  • 低到高直接自动转换，高类型的高位自动填充0即可
  • 低到高需要截断，根据类型的长度来从低位开始截断，即强制类型转换

存储顺序

在内存中，数据都是按内存地址顺序从低到高存储的

字节顺序是指占内存多于一个字节类型的数据在内存中的存放顺序——百度百科

为什么会产生存储顺序问题，之前一直没搞懂，今天有了一点想法，似乎因为对内存的访问是通过地址来实现的，比如访问一个int类型，有4个字节，我们访问的是int类型的首字节地址，在int这个四字节“unit”中，实际数值在每个字节中如何放置，这里就产生了数据问题，即小端法还是大端法

比如一个int类型变量，有4个字节长度，数值为0x12345678，分配给变量的内存地址为0x01 0x02 0x03 0x04

（我们访问变量，其实就是访问0x01，然后根据变量的类型，给我们连续4个字节的值）

小端法：数值低字节放在内存地址低字节
大端法：数值高字节放在内存地址低字节

内存地址 0x01 0x02 0x03 0x04
小端法 0x78 0x56 0x34 0x12
大端法 0x12 0x34 0x56 0x78

查看内存

gdb中用X指令来查看内存

x/[n/f/u] addr

n、f、u是可选参数，addr是要查看的内存地址

  • n：显示几个units
  • f：格式，具体可选：
    • x：按16进制格式显示，是默认值
    • d：按10进制显示
    • u：按无符号10进制显示
    • o：按8进制显示
    • t：按二进制显示
    • i：指令显示
    • s：字符串显示
    • a：16进制格式显示地址
    • c：按字符串显示
    • f：按浮点数显示
  • u：显示unit的大小，即将内存按unit块一个一个显示，unit块中用了小端法
    • b：单字节
    • h：双字节
    • w：四字节
    • g：八字节

addr是内存地址，可以用直接地址，如0x8048000，或者用寄存器，如$esp,$esp+0x10

常用命令：x/30xw $rsp

linux命令-4-ln

Posted on 2019-04-08

ln命令可以在不同的目录下使用同一个文件或目录，不需要把文件或目录复制过去，只需要建立一个链接

命令格式

ln [参数] [源文件或目录] [目标文件或目录]

软连接

  • 特点

    • 以路径的形式存在，类似于windows的快捷方式
    • 可以跨文件系统
    • 可以对不存在的文件名进行链接
    • 可以对目录进行链接

ln -s [源] [目标]

硬链接

  • 特点

    • 以文件副本的形式存在，但不占用实际空间
    • 不可以给目录链接
    • 只能在同一个文件系统下

ln [源] [目标]

其他参数

  • -b 删除

aircrack破解wifi

Posted on 2019-03-31

aircrack-ng是处理无线的一个工具包，我们使用的是：

  • airmon-ng
  • airodump-ng
  • aireplay-ng
  • aircrack-ng

现在要做的是破解一个wifi密码，大体过程为：

  1. 配置网卡
  2. 探测wifi信息
  3. 抓包
  4. 破解

配置

用的环境是虚拟机kali，外面接了个ALFA网卡

  1. 查看网卡

    执行ifconfig,找到了网卡 wlan0

  2. 开启网卡

    ifconfig wlan0 up

  3. 将网卡变成监听模式

    airmon-ng start wlan0

    然后发现网卡变成了wlan0mon，即监听模式的网卡，此时网卡配置完成

探测wifi信息

airodump-ng wlan0mon

抓包查看信息，在ESSID中找到要破解的wifi名字，记下对应的MAC地址(BSSID)和频道(channel,CH)

抓包

为了破解wifi，需要抓取对应wifi的握手包

airodump-ng --bssid 地址 -c 频道 -w 存储文件名 wlan0mon

这时候要等有设备连接wifi才会抓到，我们是不会等的，可以用拒绝服务攻击来强制让设备断开连接，不停的重连

新开一个shell

aireplay-ng -0 0 -a 地址 wlan0mon

  • -0表示用接触认证攻击，后面的参数0表示不断发送攻击数据包
  • -a表示AP的MAC
  • -c参数可选，表示要被攻击的客户机MAC，不写的表示所有设备

此时看到抓包界面的右上角出现handshake：地址，即抓包完成了

破解

对抓到的包用字典进行破解

aircrack-ng -w 字典 数据包.cap
